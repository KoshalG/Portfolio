Some good HTML practices


HTML is the language of the web. It is the foundation of every website and web application. It is the first thing that you need to learn when you start your journey as a web developer. It is a very simple language to learn and understand. But there are some good practices that you should follow while writing HTML code. In this blog, I will be discussing some of these practices.


Always Declare a Doctype


Doctype stands for Document Type Declaration. It is an instruction to the web browser about what version of HTML the page is written in. It is the first thing that you should write in your HTML document.


Always use UTF-8 as the character encoding


UTF-8 is the preferred encoding format for HTML5 documents. It is the most widely used encoding format. It supports all the characters of all the languages in the world. It is also the default encoding format for HTML5 documents. So, you don't need to specify it explicitly. But it is a good practice to specify it explicitly.


Always use lowercase for tags and attributes


HTML is not case sensitive. So, you can use uppercase or lowercase for tags and attributes. But it is a good practice to use lowercase for tags and attributes. It makes your code more readable and maintainable.


Always use double quotes for attributes


HTML allows you to use single quotes or double quotes for attributes. But it is a good practice to use double quotes for attributes. It makes your code more readable and maintainable.
